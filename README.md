# InfotecsTestTask

## Обзор
Это тестовое задание на стажировку в компании ИнфоТекс.

## Краткое описание задания
### Часть 1
Разработать библиотеку для записи текстовых сообщений в журнал. В качестве журнала использовать текстовый файл.

### Часть 2
Разработать консольное многопоточное приложение для проверки библиотеки записи сообщений в журнал.

## Запуск
Сборка:
```
make
```
Запуск приложения:
```
make run
```
Запуск Unit тестов\
Примичания:
* В задании не было указано обязательное написание Unit тестов, это прерогатива автора, точно так же как и документация;
* Для их запуска необходимо иметь установленный фреймворк GTest.
```
make test
```

## Краткое описание работы приложения
При запуске программе в параметрах командной строки можно ввести путь к файлу журнала и уровень важности сообщения по умолчанию.

Если эти данные не будут введены, то после страта программа попросит ввести путь к файлу журнала, а уровень важности установиться автомотически.

Файл журнала создаётся относительно директории запуска (при запуске с помощью `make run` файл создастся в папке `./Program`).

При запуске программы создаётся побочный поток, который занимается записью сообщений в журнал при помощи библиотеки из первой части.

При вводе сообщения, сообщение помещается в очередь и оповащется побочный поток, после чего, побочный поток записывает сообщение в файл.
